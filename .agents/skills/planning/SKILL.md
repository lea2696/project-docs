# Planning Skill

Convert goals into implementation plans for the project-docs framework.

## Before Planning

Read: `product/VISION.md`, `product/SCOPE.md`, `product/ROADMAP.md`, `architecture/SYSTEM_OVERVIEW.md`, `context/CONVENTIONS.md`, `context/CURRENT_STATE.md`

## Output (use plans/PLAN_TEMPLATE.md)

1. Objective
2. Included/Excluded Scope
3. Risks
4. Affected Areas
5. Ordered Steps
6. Human Checkpoints
7. Validation Strategy
8. Documentation Updates
9. Done Criteria

## Constraints

- Do not modify any files. Analysis and planning only.
- If ambiguous, pause and ask the human.
