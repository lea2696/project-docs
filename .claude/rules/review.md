# Review Rules

## Rejection Conditions

Reject work that:
1. Changes behavior without updating README (both EN and ES)
2. Introduces unrelated refactors into a scoped task
3. Breaks scaffold/dogfood structural alignment
4. Adds agent roles without updating both scaffold and dogfood versions

## Approval Requirements

- All planned changes are implemented
- Scaffold and dogfood versions are structurally aligned
- README EN/ES sections are parallel
- No unrecorded scope expansion
